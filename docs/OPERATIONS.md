# Rust 版操作指南

本文档面向日常使用和运维，覆盖当前 Vue 3 前端与 Rust API 的主要操作。

## 登录与会话

首次启动需要在环境变量中设置：

```env
TOKEN_ENCRYPTION_KEY=<base64 32 字节密钥>
ADMIN_USERNAME=admin
ADMIN_PASSWORD=<至少 8 位的初始密码>
```

登录成功后，后端设置：

- `session_id`：HttpOnly 会话 Cookie。
- `csrf_token`：前端读取后用于 `X-CSRF-Token` 请求头。

前端会自动处理 Cookie 和 CSRF。手动调试 API 时，`POST` / `PUT` / `DELETE` 必须携带 `X-CSRF-Token`。

键盘用户可在页面首次聚焦时使用“跳到主要内容”入口跳过导航，焦点会移动到当前主内容区域。

登录后使用左侧响应式导航切换账户、记录、统计、通知、全局设置和用户管理；窄屏设备会自动改为顶部横向导航。页面的登录入场、卡片滚动显现和标题滚动反馈由 GSAP 驱动，系统启用“减少动态效果”时会自动降级为静态界面。

## 账户管理

账户页支持：

- 按用户、站点类型、启用状态、签到状态筛选。
- 按关键词搜索账户名称、站点地址、备注。
- 多选账户。
- 批量签到、批量刷新余额、批量启用、批量禁用。
- CSV 导入和导出。

编辑账户时：

- 站点类型和认证方式不可修改。
- `accessToken` / `cookie` 留空表示保持原值。
- `userId`、`customCheckinUrl`、`note` 留空表示清空为 `NULL`。

## 批量签到

批量签到入口：

- 当前列表签到。
- 某个用户分组签到。
- 选中账户签到。
- 签到记录页失败账户重试。

后端执行策略：

- 先批量查询账户和今日次数，避免 N+1。
- 普通用户只能执行自己的账户。
- 自动跳过禁用账户、今日已签账户、关闭重试账户、达到每日上限账户。
- 剩余账户打乱顺序后串行执行。
- 相邻账户之间按 `batchDelayMin` / `batchDelayMax` 随机等待。

批量响应包含：

```json
{
  "items": [],
  "total": 0,
  "succeeded": 0,
  "skipped": 0,
  "failed": 0
}
```

## 签到记录

签到记录页支持：

- 状态筛选：成功、失败、已签、进行中。
- 触发方式筛选：手动、批量手动、定时。
- 时间范围筛选。
- 账户筛选。
- 失败记录重试。
- 当前列表失败账户批量重试。
- 复制摘要。
- 清理记录。

清理记录时：

- `keepLatest > 0`：在当前清理范围内保留最新 N 条。
- `keepLatest = 0`：清空当前范围内的全部签到历史。
- 普通用户只能清理自己的记录；管理员可清理当前筛选用户，未筛选用户时清理全局记录。
- 清空全部时可勾选“同时重置最近签到状态和失败计数”；该操作保留账户余额和余额刷新时间。
- 清理是 SQLite 逻辑删除，不会在请求中执行 `VACUUM`，数据库文件大小可能不会立即缩小。

## 数据统计

统计页包含：

- 总账户数、启用率。
- 今日成功、今日失败、今日失败率。
- 区间成功率和总执行次数。
- 总余额。
- 每日签到趋势。
- 站点统计。
- 最近失败。
- 风险站点摘要。

管理员可按用户筛选；普通用户即使传入其他 `userId`，后端也会拒绝。

## 通知设置

支持通知类型：

- email
- webhook
- telegram

触发条件：

- 签到失败通知。
- 连续失败阈值。
- 余额过低通知。
- 余额阈值。

通知页提供本地校验、目标预览、触发条件预览和测试结果留痕。

更新通知时以下字段留空可清空：

- `balanceThreshold`
- `webhookHeaders`
- `note`

## 全局设置

仅管理员可访问。

字段说明：

| 字段 | 说明 |
|------|------|
| `enabled` | 是否启用自动签到 |
| `windowStart` / `windowEnd` | 签到窗口，本地时间，格式 `HH:MM` |
| `retryEnabled` | 是否启用失败重试 |
| `maxAttemptsPerDay` | 每账户每日最大尝试次数 |
| `batchDelayMin` / `batchDelayMax` | 批量和定时签到的账户间随机延迟 |
| `cleanupKeepLatest` | 定时清理保留最新记录数，`0` 表示清空全部 |

边界：

- `maxAttemptsPerDay`：`1~100`
- `batchDelayMin` / `batchDelayMax`：`0 <= min <= max <= 600`
- `cleanupKeepLatest`：`0~10000`

## 推荐验证流程

功能变更后建议执行：

```bash
cargo fmt -- --check
cargo test
cargo clippy -- -D warnings
cd frontend && npm run build
```

涉及前端或 API 行为时，至少手动验证：

- 登录、刷新页面会话保持、退出。
- 账户创建、编辑、清空可选字段、删除。
- 单账户签到、批量签到、失败重试。
- 设置保存和非法值提示。
- 通知创建、编辑、清空可选字段、测试发送。
- 普通用户无法访问其他用户数据。
